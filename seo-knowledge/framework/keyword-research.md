### 🧩 合并原则（任何文档整合都适用）

> **合并时只看"内容是否有用"，不看"有几份文档"。**
> 不论源文档数量多少，只萃取对目标有用的部分；无价值、重复、或过时的内容一律不纳入——
> 宁缺毋滥，绝不为了"凑齐来源"而保留无用段落。合并的目标是**更精炼可用的产物**，不是保留全部原文。

**"有用"的操作定义（一段内容须全部满足才纳入）：**

| # | 测试 | 通过条件 | 不通过（直接丢弃或只摘子句） |
|---|------|---------|------------------------------|
| 1 | **可执行性** | 读后让人改变做法——新增了步骤 / 清单 / 阈值 / 模板 / 决策规则，且目标文档中尚未有 | 泛泛而谈（如"SEO 很重要"）、不带来任何新动作 |
| 2 | **准确·时效** | 不被 KB 内更新/更权威的来源推翻；引用工具/数据若已失效须标注 | 已被更新来源推翻，或引用死工具/过期数据且无说明 |
| 3 | **非冗余** | 目标文档中未以同等或更强形式出现过 | 与文档已有内容重复（同义改写也算冗余） |
| 4 | **在范围内** | 符合目标文档声明的范围；类型专属内容归对应 `seo-xxx` 技能 | 把 B2C 专属打法塞进通用文档、或越界到无关主题 |
| 5 | **具体可验证** | 含具体指标 / 实例 / 反例警示（如"每组 5–8 词"、"新站 KD<20"） | 抽象无信息量、无可核对的事实 |
| 6 | **增量深度** | 若同主题已存在，新内容提供**更详细的解读或更强可落地的指导**（更细的步骤、更多示例、更清楚的因果）→ 视为升级，替换旧版而非丢弃 | 与已有内容同质、未增加任何新信息或更深解读（此时按 #3 判为冗余） |

> 只要有一条不通过 → 不纳入；若段落里只有一句有用，则**只摘那一句**，不保留整段。
> 第 6 条是对"冗余"的补充：同样是讲某个点，新来源若比现有**讲得更透、指导更具体**，就是"新增的知识"而非重复——用新版替换旧版，不留两份。

## 一、核心概念

### 1.1 搜索意图 4 类

| 意图 | 搜什么 | 转化率 | 对应内容 | 例子 |
|------|--------|--------|---------|------|
| **交易型** | vs / best / review / pricing / buy | 10–20% | 对比页 / 替代品页 / 产品页 | "tattoo studio pro alternatives" |
| **商业型** | software / app / system / tool | 6–10% | 功能页 / 产品页 | "tattoo booking software" |
| **信息型** | how to / guide / what is | 0.5–2% | 博客 / 指南 | "how to digitize tattoo waivers" |
| **导航型** | 品牌名 / 产品名 | 极高（需品牌认知） | 品牌页 | "inkflow" |

**快速判断意图**：搜这个词 → 看 Google 首页结果类型：
- 全是 Landing Page / Product Page → 商业型
- 全是 Blog / Guide → 信息型
- 全是 Compare / vs → 交易型
- 全是品牌页 → 导航型

**原则**：前期 80% 精力做商业型 + 交易型，20% 做信息型。

### 1.2 竞争度 KD 判断

**方法 A：用工具（精确）**

| 工具 | 费用 | KD 参考值 | 备注 |
|------|------|----------|------|
| Google Keyword Planner | 免费（需 Ads 账号） | 无 KD 分，看建议出价 | 出价高 = 竞争高 |
| Ahrefs | $29/月起 | 0–100 分 | 行业标准 |
| Semrush | 免费版有限 | 0–100% | 算法与 Ahrefs 不同 |
| Ubersuggest | 免费有限 | 0–100 分 | 不太精准但够用 |

> KD 分只是参考，**不是绝对标准**。Ahrefs 的 KD 算法偏向 DA（域名权重），内容比对手好 10 倍，DA 低也能排。

**方法 B：手动检查表（免费，不花钱）**

打开 Google 搜这个词，看首页结果：

```
✅ 低 KD（可以做）：
  □ Reddit / Quora / 论坛排首页
  □ 排在前面的页面是博客文章，不是产品首页
  □ 只有 0–2 个广告
  □ 搜索结果量 < 5000 万条
  □ 内容过时/一般的文章还在前排

❌ 高 KD（暂时别碰）：
  □ Forbes / HubSpot / Shopify 等大站排首页
  □ Wikipedia 在首页
  □ 结果全是 Landing Page
  □ 4+ 个广告
  □ 搜索结果量 > 1 亿条
```

> 新站 KD < 20 起步，20–40 有外链后上。

### 1.3 五维度选词评分系统（量化排序用）

每个词按 5 项打分（1–10），总分 50，≥35 优先做：

| 维度 | 权重 | 说明 |
|------|------|------|
| **KD（竞争度）** | ⭐⭐⭐⭐⭐ | 当前 DR 能不能做？0–15 起步，15–30 积累期 |
| **搜索意图** | ⭐⭐⭐⭐⭐ | 商业/交易型 > 信息型。搜这个词的人想买还是想学？ |
| **商业价值** | ⭐⭐⭐⭐ | 带来转化的概率。对比页 > 功能页 > 博客 |
| **搜索量** | ⭐⭐⭐ | 太低没流量，太高做不上去 |
| **内容匹配度** | ⭐⭐⭐ | 能不能自然融入已有内容结构？ |

```
举例：digital tattoo consent form app
  KD 5-10        → 9 分（非常低）
  商业型          → 8 分（用户在找产品）
  商业价值高      → 8 分（直接功能匹配）
  月搜索量 200-500 → 6 分（中等）
  内容易匹配      → 9 分（正好有 Waivers 功能页）
  总分 40/50 ✅ 做
```

## 二、三阶段调研闭环（核心操作流程）

来源：微信《独立站谷歌SEO关键词调研完整流程》+ 上述概念。**这是把关键词从"想法"变成"页面"的主流程。**

### 阶段一 · 挖词（词根 → 上千候选词）

1. **找词根**（投入产出比最高的一步，多花两小时可避免后续几十小时无效扩展）：
   - ① **问人**：产品经理 / 老板 / 行业供应商——海外买家怎么称呼？一个产品英文可能有多种叫法（colored / cosmetic / fashion contact lenses）。
   - ② **Google Autocomplete**：输入框 A→Z 各跑一遍，收集下拉提示词（真实搜索行为）。
   - ③ **GSC "搜索结果"报告**：导出网站已产生曝光/点击的查询词（Google 认为相关的词）。
2. **SEMrush Keyword Magic Tool 扩展**：每个词根查询 → 导出 → 仅留 **关键词 / Volume / KD / Intent** 四列 = **表A**。
   - 多 CSV 合并（Windows PowerShell）：
     ```
     Get-ChildItem -Filter "*.csv" | ForEach-Object { Import-Csv $_.FullName } | Export-Csv -Path "merged_keywords.csv" -NoTypeInformation -Encoding UTF8
     ```
3. **竞品反挖**：Google 搜核心词，前 3 页中除 Amazon/Alibaba 外的独立站 = 竞品（找 30–50 个）；用 SEMrush **Organic Research** 导出每个竞品排名词 = **表B**（列序对齐表A）。
4. **表A + 表B 合并去重 = 表C**（完整词库，可能几万词）。

```
词根收集（问人+GSC+Autocomplete）
  ↓ SEMrush Keyword Magic Tool
表A（词根扩展）
  ↓ + 竞品 Organic Research
表B（竞品反挖）
  ↓ 合并去重
表C（完整关键词库）
```

### 阶段二 · 分组（几万词 → 上百簇）

1. **筛选**（先复制备份原表C，所有操作在副本上）：
   - ① 删搜索量 = 0 的词。
   - ② 删别人品牌词 & 不相关品类词（如卖喂食器却出现 PetSmart / 玩具词）。
   - ③ 按品类设下限阈值：大品类（服装/电子）≥1000，小众品类（工业设备）可低至 50–100——**看表C 前 50 词量级定，别拍脑袋**。
2. **按词义聚簇**（每组 5–8 个相近词）：单复数/词性变体、同义词替换、介词混用或缺失、修饰词增减（best/reviews 可归同组）。
3. 人工分组最准（500 词内）；超几千词用 Cluster Keywords / Keyword Cupid（按 SERP 重叠度自动分，再人工微调）。

### 阶段三 · 页面规划（簇 → 页面，闭环关键一步）

每个关键词簇对应一个网站页面。映射规则：

| 页面类型 | 意图 | KD | 每页词数 | 示例 |
|---------|------|----|---------|------|
| 首页 | 品牌 + 品类大词 | 高（50+） | 3–5 核心词 | pet supplies |
| 类目页 | Commercial / Transactional | 中（20–50） | 5–15 | automatic pet feeder |
| 产品页 | Transactional | 低（10–30） | 3–8 精准长尾 | 5L wifi stainless steel dog feeder |
| 博客 | Informational | 低–中（10–40） | 5–10 信息词 | how to choose automatic pet feeder |

**4 条总原则**：
1. **一个簇对应一个页面**（两簇语义不同别放同页，否则 Google 主题困惑）。
2. **搜索意图决定页面类型**：Commercial/Transactional → 类目/产品页；Informational → 博客。
3. **难度高的词放权重高的页**：新站博客难竞争 KD>50 词，高难词放首页/类目页借权重；低难长尾放博客/产品页。
4. **避免关键词蚕食**：两页簇高度相似 → 合并或明确区分意图。

## 三、竞品关键词差距分析

```
1. 找出 3 个主要竞品（如 Tattoo Studio Pro / Tattogenda / Booksy）
2. 搜竞品排的词：site:competitor.com 看页面，或用 Ahrefs / AITDK 插件 Traffic Analysis
3. 找"他们排了、你没做的词" → 你的机会
```

> 工具：AITDK 插件 → 打开竞品站 → Traffic Analysis 看 top keywords → 直接抄有流量你没做的词。

## 四、评分排序与月度审计

表C 筛词后，用 1.3 五维评分量化排序，并进入持续维护：

```
Step 1: 列出当前已做的关键词 → 看哪些排上/没排，GSC 查哪些带来流量
Step 2: 找新词机会 → AITDK 看竞品新增词 / Reddit 行业版块 / Google 相关搜索 / 竞品博客新文
Step 3: 按 5 维打分排序 → KD 0–15 排最前，商业/交易型优先
Step 4: 分配到内容日历 → 对比页 > 功能页 > 博客（BOFU 优先）
```

> **维护节奏**：关键词搜索量/KD 动态变化，建议每 **3–6 个月**更新一次词库（补充新词、清理失效词）。调研不是一次性工作，是持续资产。

## 五、常见坑

1. **词根只想到三五个**就开始扩展 → 多花两小时找词根，产出远高于后面写文章。
2. **只看搜索量不看意图** → 1 万搜索量的信息词放产品页转化极低；500 搜索量的交易词放产品页持续出单。
3. **分组太粗或太细** → 太粗（语义不同的词一组）主题不聚焦；太细（每组 1–2 词）页面爆炸、内容单薄。合理 5–8 词/组。
4. **调研做一次就不管** → 竞品进入 / 新趋势会改 KD 和词，需定期更新。

## 六、类型差异速查（共性之外看专属）

| 类型 | 关键词侧重 | 专属杠杆 | 详见 |
|------|-----------|---------|------|
| **SaaS** | 功能词 / 对比词 / 替代品词 / 痛点词 | BOFU 优先（对比/替代/功能页 > 博客）；免费工具引流 | `seo-saas` |
| **B2C 货品** | 产品词 / 购买意图词（best / buy / X vs Y） | **品类关键词矩阵覆盖**（材质/尺寸/风格/功能/场景各建独立页） | `seo-b2c` |
| **B2B 供应商** | 长尾规格词 "X manufacturer" / "X supplier" | 行业目录 / 商业平台（Alibaba/ThomasNet） | `seo-b2b`（待建） |
| **内容站** | 低 KD 信息长尾 | 靠量取胜，批量/聚合页 | `seo-content-site`（待建） |

> 类型专属关键词打法（如 B2C 品类矩阵、SaaS 替代品词库）的完整版在各 `seo-xxx` 技能，不在此通用文档展开。
