## 核心内容

### LinkedIn 开发客户核心逻辑
> LinkedIn 开发客户，核心不是"发产品"，而是**"建立信任+找到触发点"**。内容分享是地基，但真正产生询盘的是：有策略的私聊 + 时机 + 客户角色判断。

### 内容与私聊的关系
> **内容是为私聊服务的**，大部分情况下不是用来等询盘的。

客户点进你主页 → 判断你是不是"靠谱供应商" → 决定回不回你。所以主页内容（公司动态、工厂背景、展会背书）扮演的是信任证明角色。

### 私聊回复率低的三原因
1. **发的是"你想卖的"不是"他此刻关心的"** — 采购/品牌方/欧洲客户对冷产品推销耐受度极低
2. **客户看了但不想现在回** — 不回复 ≠ 没兴趣。外贸 LinkedIn 真实路径：先被记住 → 后面某天再用到你
3. **没有区分客户角色** — 不同角色回复率差异巨大

### 客户角色区分
| 角色 | 策略 | 是否适合一上来发产品 |
|------|------|-------------------|
| 采购/Sourcing | 非常克制 | ❌ |
| 市场/产品经理 | 不要直接发 | ❌ |
| 创始人/老板 | 讲价值，不是参数 | ❌ |
| 批发商/Distributor | 可以较早发 | ✅ |

### 3 种有效触达方式

**方法 1：轻互动 → 再私聊（回复率最高）**
```
客户发过动态 → 先点赞/留一句专业评论 → 隔 1-2 天再私聊
私聊目的：让你不再是"陌生推销者"
```

**方法 2：用"问题"替代"产品"**
```
❌ Here is our new xxx, please check.
✅ Hi Tom, for 2026 are you focusing more on entry-level or mid-range xxx?
```

**方法 3：场景型触达**
结合展会 + 工厂 + 实际项目，用行业视角而非产品硬广

### 产品发送节奏
```
关系 → 对话 → 场景 → 产品
```
先征求同意："Would it be ok if I send you a brief spec?"
回复率和接受率明显提高。

### 关键规则
1. 私聊第一条不要带附件/视频
2. 私聊信息不宜太长
3. 多用 "Curious to know…" / "May I ask…"
4. 不回的人 7-10 天后二次轻跟进（不是重复）
5. 把 LinkedIn 当作"长期客户池"，不是即时询盘工具
6. 每个客户做好跟进记录/备注
