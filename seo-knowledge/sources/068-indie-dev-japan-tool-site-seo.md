## 核心内容

### 站点数据
- GSC 近 3 个月：1,274 点击 / 7.14 万曝光 / CTR 1.8% / 平均排名 8.1
- 站点从 70 多页扩到 700 多页
- Blog 从 0 做到 20 多篇
- Google 搜索影响显示 1.2K

### 7 步增长路径

**Step 1：先搭数据底座**
> SEO 不是内容问题，而是搜索、点击、阅读、转化连在一起的问题。
- GSC → 搜索曝光、关键词、点击
- Clarity → 热力图和录屏
- GA → 用户路径和转化漏斗

**Step 2：列出搜索意图，不是只列关键词**
同一个关键词，意图不同（了解/比较/操作/排错/购买），页面结构就不同。

**Step 3：做一个长尾内容池**
> 长尾的本质，是把一个大需求拆成很多具体问题。
从日语 Google 搜索结果、竞品 Blog、竞品 FAQ、论坛问题、用户评论里找题。
AI 辅助整理，但不当最终决定者。

**Step 4：找可结构化对象**
> 只有对象能结构化，页面量才有意义。
工具站可结构化的对象：国家/城市/职业/文件格式/模板类型/平台/错误码/品类/使用场景。

**Step 5：做 20 篇高质量 Blog，再扩程序化页面**
Blog 用来解释和建立信任，程序化页面用来覆盖长尾入口。两者不要互相替代。

**Step 6：用 Clarity 和 GA 把 CTA 放到用户真正停留的地方**
> SEO 带来的只是入口。入口之后，还要看用户怎么走。
用户在哪里被说服，CTA 就应该在哪里出现。

**Step 7：每两周复盘一次**
GSC 曝光/点击/CTR/排名 → GA 注册/付费漏斗 → Clarity 停留/点击热区 → 决策下轮动作。

### 日本市场关键洞察
- **真实本地化，不是翻译**：先看日语 Blog 和搜索结果 → 观察日本用户关心什么 → 再让 AI 辅助初稿 → 人工做日语修改
- 日本用户会通过一篇篇内容建立信任，不是看一个 landing page 就下单
- 日本站的内容需要：使用场景、选择标准、对比文章、注意事项、常见问题、细节说明、本地表达
- AI 可以帮忙生成初稿，但最后一定要回到人的判断

### 程序化 SEO 的底层逻辑
> 关键在于"找到可以结构化的内容，然后用 AI 辅助，多语言一叠，页面量就上去了。"

实操方法（原文案例）：
```
卖西瓜 → 做西瓜图鉴 → 一个西瓜品种一个页面
→ 每个页面有固定模板，但内容字段不同
→ 用内链、Blog、比较页、说明页串起来
```

### 工具站页面网络结构
```
一个核心功能页
一组使用场景页
一组问题解答页
一组对比页
一组地区/语言/对象页
一组 Blog 教程页
（页面之间互相解释、互相导流 → 最后带回转化页）
```
