## 分类

## 核心发现

### 1. 内容 4 分法架构
Okara.ai 的内容按用户意图分 4 条线：

| 线 | URL 模式 | 目标意图 |
|---|---|---|
| Product Docs | `/docs/introduction`, `/docs/channels/overview` | 产品评估期（DO 意图） |
| Blog | `/blog/comparison-pages-that-rank` | TOFU 流量 → MOFU 转化 |
| Pricing | `/docs/plans` 对比定价 | BOFU 决策 |
| Product as Content | AI 工具产出即是内容 | 全漏斗覆盖 |

### 2. 文档化 = SEO 引擎
- Docs 不是枯燥 API 手册，每页可独立回答一个用户真实问题
- FAQ 页、Setup 页、Configuration 页都是 SEO landing page
- 对 SaaS 站来说，Docs 的 SEO 价值常被低估

### 3. 对比页 = 最高 ROI
- 自己写了 "How to Build Comparison Pages That Rank"
- 建议：明确承认 competitor 优点来 build trust（"Honesty as strategy"）
- 对比页提升 AI 引用率（GEO 角度）

### 4. GEO 意识
- 结构化数据 + 可引用句（拎出来独立可读的句子）
- SEO tool 自带 GEO score 跟踪（ChatGPT/Perplexity/Claude/Google AI Overviews）

### 5. 全自动审计循环
- Daily audit → fix suggestion → automated PR (via Coding Agent)
- 子页面级审计（paid plan）
