## 来源

- **日期**: 2026-06-30

## 分类
内容策略, 内容架构, pSEO, GEO

## 核心发现

### 1. 「单页 Playbook ≠ 博客」
博客按时间排列内容，用户随机点一篇；Playbook 按用户行动阶段排列内容，用户按顺序走完整个流程。

### 2. 站点架构
```
├── #route         → 流程地图（7步卡片一览）
├── #playbook      → 详细实战计划（6大阶段）
│   每步含：做什么 + 工具 + 验证标准 + 产出物
├── #starter-path  → 新手视频路线（8步，每步配3-4个视频）
├── #tools         → 工具速查（4分类）
├── #checklist     → 7天行动计划
└── #growth        → 增长飞轮
```

### 3. 关键设计模式
- **Pass/Fail 标准**：每个步骤明确「做到什么程度算通过」，不是空谈
  - 例如：12%-25% 访问者到付费墙 = 需求验证通过
  - 例如：用户不用解释也能完成核心动作 = MVP 通过
- **外部资源做补充而非主干**：YouTube 视频链接作为 each section 的 deeper dive
- **模板/清单/案例标签**：供用户快速评估内容密度
- **777 行单页**，纯静态 HTML+CSS+JS，无框架负担

### 4. 产品即内容
- 这个站本身就是一个产品（Playbook），不是工具的附属 blog
- 内容组织围绕「用户的学习路径」，不是「产品的功能列表」

## 对 InkFlow 的适用性
**学模式不学格式：**

| 学什么 | 不学什么 |
|---|---|
| ✅ 判断标准（pass/fail criteria） | ❌ 不做单页 700 行怪兽 |
| ✅ Checklist 清单式结构 | ✅ 拆成多篇 focused 页面 |
| ✅ 按用户行动阶段组织 | ✅ 保持 feature pages 深度 |

⚠️ **受众差异**：App-Playbook 的受众是独立开发者（喜欢读长文），InkFlow 的受众是纹身师（忙、视觉型）。同模式但需换格式。
