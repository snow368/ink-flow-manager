## 三、内容更新策略（常更新怎么做）

### 为什么需要更新
Google 奖励新鲜内容。一篇内容发布后流量曲线：
```
发布时间 → 第 1 周（爬取）→ 第 2-4 周（排名爬升）→ 第 2-3 月（峰值）
  → 第 4-6 月（开始衰减）→ 第 6 月+（需要刷新）
```

### 什么内容要更新

| 类型 | 更新频率 | 触发条件 |
|------|---------|---------|
| **博客** | 每 6-12 个月 | 数据过时 / 排名下降 / 新增功能 |
| **对比页** | 每 3 个月 | 竞品改价格 / 改功能 |
| **功能页** | 每 6 个月 | 产品功能更新 |
| **Landing Page** | 每 3 个月 | 转化率下降 |
| **FAQ** | 每月检查 | 新增常见问题 |

### 更新流程
```
GSC 发现排名下降
  → 检查内容是否过时
    → 补充新数据/新截图
    → 更新内链
    → 改 updatedDate
    → 重新提交 GSC 索引
    → X/Twitter 重新推广
```

### 每月固定的更新任务
```
第 1 周：检查上月发布的博客排名 → 排名下降的更新
第 2 周：检查所有对比页 → 竞品有变化就改
第 3 周：FAQ 过一遍 → 加新问题
第 4 周：Landing Page 数据 → CTA 优化
```
