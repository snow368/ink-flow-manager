当用户请求审核 app 时（说"审核"、"/audit"、"检查质量"等），按以下流程执行：

1. 加载 `.claude/audit-standards.md` 中的四级标准
2. 逐条检查代码库，按类别（A→B→C→D）依次审核
3. 生成报告，格式如下：
   - 每项标记 🔴严重 / 🟡警告 / 🔵建议 / ✅通过
   - 引用具体文件名和行号
   - 简要说明问题原因和修复方向
4. 报告末尾给出汇总分数（通过数/总检查项）和 Top 3 优先修复项
5. 重点区域：
   - 所有 tsx 文件中硬编码的英文字符串（i18n 缺口）
   - `as any` 类型断言
   - 安全敏感操作（innerHTML、eval、API key 硬编码）
   - 颜色/间距/圆角未使用 THEME 常量
